## Getting Started

setting up local environment.

<Steps>

<Step>

### Fork the repository

Go to the [Aura Auth repository](https://github.com/aura-stack-js/auth) and click the "Fork" button in top right.

</Step>

<Step>

### Clone your fork

Clone and configure fork. [Contributing to a project](https://docs.github.com/en/get-started/exploring-projects-on-github/contributing-to-a-project)

```bash
git clone https://github.com/YOUR_USERNAME/auth.git
cd auth
```

</Step>

<Step>

### Add upstream remote

Configure a remote repository for a fork [GitHub docs](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/configuring-a-remote-repository-for-a-fork).

```bash
git remote add upstream https://github.com/aura-stack-ts/auth.git
```

</Step>

<Step>

### Install dependencies

Use [pnpm](https://pnpm.io/) package manager to install the dependencies.

```bash
pnpm install
```

</Step>

</Steps>

## Contribute

Once you know what you want ton contribute, it's time to make changes.

<Steps>

<Step>

### Create a new branch

```bash
# syncronize your repo
git switch master && git pull upstream master

# create a new branch to work in.
git switch -c feature/your-feature-name
```

</Step>

<Step>

### Make your changes

Make your changes in the development environment

Packages server:

```bash
pnpm dev
```

Doc server

```bash
pnpm dev:docs
```

</Step>

<Step>

### Write or update tests

Ensure your changes are covered by tests:

```bash
# Run tests
pnpm test

# Run tests with coverage
pnpm test:coverage
```

</Step>

<Step>

### Code style

Verify the type chekhing

```bash
pnpm type-check
```

Format codebase

```bash
# format code
pnpm format

# check formating
pnpm format:check
```

</Step>

<Step>

### Commit your changes

Follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:

```bash
git commit -m "feat(auth): add support for OAuth 2.0 PKCE flow"
```

Push to your fork

```bash
git push origin feature/your-feature-name
```

Open a pull request

Go to GitHub and open a [Pull Request](https://github.com/aura-stack-ts/auth/pulls) with:

- Clear and descriptive title
- Reference to related issues
- Comprehensive description of changes
- Screenshots or GIFs for UI changes (if applicable)

</Step>

</Steps>

## Development Workflow

### Project Structure

```
auth/
├── packages/
│   ├── core/           # Main authentication library
│   │   ├── src/        # Source code
│   │   └── test/      # Tests
│   └── jose/           # JOSE utilities
│       ├── src/        # Source code
│       └── test/      # Tests
├── docs/               # Documentation site (Fumadocs)
├── apps/               # Example applications
└── turbo.json          # Turborepo configuration
```

### Running Tests

```bash
# Run all tests
pnpm test

# Run tests in watch mode
pnpm test:watch

# Run tests with coverage
pnpm test:coverage
```

### Best practices

- Use TypeScript for all new code
- Write meaningful variable and function names
- Add TsDoc comments for public APIs
- Keep functions small and focused
- Write tests for new features
- Follow existing code patterns

Your contributions help make authentication easier and more secure for developers everywhere.

### Need help ?

Unsure where to begin? Look for issues labeled:

- Open an [issue](https://github.com/aura-stack-ts/auth/issues) with questions and look issues.
- `good first issue` - Issues that are good for newcomers
- `help wanted` - Issues that need assistance

Thank you for contributing to Aura Auth.
