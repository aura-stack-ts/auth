## Environment Variables

Aura Auth requires environment variables to store sensitive data related to OAuth credentials, secret keys for signing and encrypting JWTs and CSRF tokens, and salts for key derivation.

### Secure Variables

| Name               | Description                                                                                           |
| ------------------ | ----------------------------------------------------------------------------------------------------- |
| `AURA_AUTH_SECRET` | 32-byte secret for JWTs signing/encryption                                                            |
| `AURA_AUTH_SALT`   | Optional but recommended salt for key derivation used for signing and encrypting JWTs and CSRF tokens |

<Callout>
  The `AURA_AUTH_SECRET` environment variable is overridden by `secret` configuration option in the `createAuth` function. On the
  other hand, the `AURA_AUTH_SALT` environment variable is not overridden by any configuration option and its recommended to set
  the environment variable to create a secure salting to sign and encrypt JWTs and CSRF Tokens.
</Callout>

### OAuth Variables

| Pattern                              | Description                               |
| ------------------------------------ | ----------------------------------------- |
| `AURA_AUTH_{PROVIDER}_CLIENT_ID`     | Client ID obtained from the OAuth app     |
| `AURA_AUTH_{PROVIDER}_CLIENT_SECRET` | Client Secret obtained from the OAuth app |

<Callout>
  All of the environment variables that follow the previous patterns are loaded automatically by Aura Auth, so there is no need to
  set the environment variables in the OAuth provider configuration.
</Callout>

Some of the supported OAuth providers provided by Aura Auth include:

- [GitHub](/docs/oauth/github)
- [GitLab](/docs/oauth/gitlab)
- [Spotify](/docs/oauth/spotify)
- [Bitbucket](/docs/oauth/bitbucket)

To see all the providers supported by Aura Auth, see [OAuth Providers](/docs/oauth).

```bash title=".env"
# GitHub OAuth Credentials
AURA_AUTH_GITHUB_CLIENT_ID=""
AURA_AUTH_GITHUB_CLIENT_SECRET=""

# GitLab OAuth Credentials
AURA_AUTH_GITLAB_CLIENT_ID=""
AURA_AUTH_GITLAB_CLIENT_SECRET=""

# Spotify OAuth Credentials
AURA_AUTH_SPOTIFY_CLIENT_ID=""
AURA_AUTH_SPOTIFY_CLIENT_SECRET=""

# Bitbucket OAuth Credentials
AURA_AUTH_BITBUCKET_CLIENT_ID=""
AURA_AUTH_BITBUCKET_CLIENT_SECRET=""
```

<Callout type="warn">Never commit your `.env` file. Always use a secret manager in production.</Callout>

## Generating `AURA_AUTH_SECRET`

```bash
# Using Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"

# Using OpenSSL
openssl rand -base64 32
```
